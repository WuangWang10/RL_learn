# $\epsilon$贪婪算法

---

## 问题构建
假设有K臂老虎机，每次拉下一根手臂会得到对应的奖励，而得到奖励的概率取决于该臂设计的奖励概率；K根手臂的概率分布恒定但未知。

我们的目标是通过不断地与多臂老虎机交互，找到获得奖励概率最大的那一根拉杆，并使得在整个交互过程中，所获得的奖励最大。

为了之后判断算法的好坏，引入懊悔和累计懊悔的概念；
对每一个动作，定义其奖励的期望为$Q(a) = \mathbb{E}_{r\sim R(\cdot|a)}(r)$，这里奖励的期望实际上是该拉杆采样的期望，反应了该拉杆获奖的概率。

由此，一定至少存在一根拉杆，其奖励的期望（获奖的概率）不小于其他任何一根拉杆。将这根拉杆获奖的期望记为$Q^{*} = max_{a\in A}[Q(a)]$

- **懊悔**：拉动当前拉杆的动作$a$与拉动最优拉杆的期望奖励差（获奖概率差），即$R(a) = Q^{*} - Q(a)$
- **累计懊悔**：拉动$T$次拉杆后累计的懊悔总量，对于一次完整的$T$步序贯决策$\{a_{1}, a_{2}, \dots, a_{T}\}$，累计懊悔表示为：$\sigma_{R} = \sum_{t=1}^{T}R(a_{t})$

增量式期望更新：
$$\begin{align*}Q_{k} =& \frac{1}{k}\sum_{i=1}^{k}r_{i}\\ =& \frac{1}{k}(r_{k} = \sum_{i=1}{k-1}r_{i}) \\ =&\frac{1}{k}(r_{k} + (k-1)Q_{k-1}) \\ =&\frac{1}{k}(r_{k} + kQ_{k-1} - Q_{k-1})\\ =& Q_{k-1} + \frac{1}{k}(r_{k}-Q_{k-1})\end{align*}$$
对应的代码框架：
- 对于$\forall a\in A$，初始化计数器$N(a) = 0$和期望奖励估值$\hat{Q}(a) = 0$

for t = 1 $\to$ T do

1. 选取某根拉杆，该动作记为a
2. 得到奖励$r_{t}$
3. 更新计数器$N(a_{t}) = N(a_{t}) + 1$
4. 更新期望奖励估值$\hat{Q}(a_{t}) = \hat{Q}(a_{t}) = \frac{1}{N(a_{t})}[r_{t} - \hat{Q}(a_{t})]$

end for

设计一个基础求解器，作为不同求解算法的部署接口。

---

## 算法原理
在引入$\epsilon$贪婪算法之前，先了解**完全贪婪算法**。

完全贪婪算法是在**每一个决策点**都采取**期望奖励估值最大**的动作。

$\epsilon$贪婪算法在完全贪婪算法的基础上添加了噪声，使得在小概率（$\epsilon$）下，可以**随机选择**一个动作，在大概率（$1-\epsilon$）下，选择估计期望奖励最大的动作。

$$a_{t} = \left\{ \begin{aligned} argmax_{a\in A}\hat{Q}(a) \\ \text{从A中随机选择} \end{aligned} \right.$$

随着小概率随机采样的次数增加，我们得到的期望奖励也越来越接近实际的概率，这时候可以适当的降低$\epsilon$的取值，使得直接取最大期望奖励的情况增加。
因此，我们可以设计随时间衰减的$\epsilon$，需要注意的是，**$\epsilon$不会在有限时间内衰减到0（逼近但不达到），否则算法得到的解在$\epsilon = 0$之后陷入局部最优**。

---

## 核心思想

平衡**探索**与**利用**

- **探索**是指不断地随机采样，使得收集到的数据量增多，对实际情况的判断更准确

- **利用**是指使用**已知领域内的最优解**，扩大优势

## 代码演示
### 多臂老虎机代码实现
多臂老虎机代码类似于强化学习中的“环境”，能与智能体交互并给出奖励反馈。
用到的库主要是`python -> numpy`。
要实现的功能：
1. 根据设定的拉杆数确定概率分布
2. 确定最大概率和对应的拉杆编号
3. 确定拉下某一拉杆后是否给出奖励
```python
import numpy as np

class Bandit:
    def __init__(self, K):
        self.K = K
        self.probs = np.random.uniform(size=K)      # 随机生成k个0~1的数，作为每根拉杆的获奖概率
        self.best_idx = np.argmax(self.probs)
        self.best_prob = self.probs[self.best_idx]

    def step(self, k):
        if np.random.rand() < self.probs[k]:    # 拉动拉杆，判断是否获奖
            return 1
        else:
            return 0
```

设计基础求解器，使之实现：
1. 选择动作
2. 获取奖励
3. 更新期望奖励
4. 计算累计懊悔

```python
class Solver:
    def __init__(self,bandit):
        self.bandit = bandit
        self.counts = np.zeros(self.bandit.K)       # counts列表记录每根拉杆的尝试次数
        self.regret = 0     # 初始化累计懊悔
        self.regrets = []   # regrets列表记录每一步的累计懊悔
        self.actions = []   # actions列表记录选择的动作序列

    def run_one_step(self):
        raise NotImplementedError
        # 单步运行取决于Solver求解器的子类策略
        # 根据子类策略选择动作k
        # 根据选择的动作k获得奖励
        # 根据获得的奖励更新期望奖励
    
    def update_regret(self, k):
        self.regret += self.bandit.best_prob - self.bandit.probs[k]     # 更新累计懊悔，最佳期望用最佳概率直接代替
        self.regrets.append(self.regret)    # 更新regrets列表

    def run(self, num_steps):
        for _ in range(num_steps):      # “_”不参与循环内运算，可以用作“i”只更新循环
            k = self.run_one_step()     # 从子类的run_one_step中得到选择的动作
            self.counts[k] += 1         #更新对应动作的计数器
            self.actions.append(k)      # 更新actions列表
            self.update_regret(k)       # 更新累计懊悔
```

$\epsilon$贪婪算法：

```python
class EpsilonGreedy(Solver):
    def __init__(self, bandit, epsilon=0.1, init_prob=1.0):
        super().__init__(bandit)
        self.epsilon = epsilon
        self.estimate = np.array([init_prob]*self.bandit.K)

    def run_one_step(self):
        if np.random.rand() < self.epsilon:
            k = np.randint(0,self.bandit.K)
        else:
            k = np.argmax(self.estimate)
        
        r = self.bandit.step(k)
        self.estimate[k] = self.estimate[k] + (1/(self.counts[k] + 1))*(r-self.estimate[k])
        return k
```

import matplotlib.pyplot as plt

def polt_results(solvers, solver_names):
    for idx, solver in enumerate(solvers):
        time_list = range(len(solver.regrets))
        plt.plot(time_list, solver.regrets, label=solver_names[idx])
        plt.xlabel('Time Steps')
        plt.ylabel('Cumulative regrets')
        plt.title('%d-armed bandit'% sovers[0].bandit.K)
        plt,legend()
        plt.show()

尝试随时间衰减的$\epsilon$

```python
class DecayingEpsilonGreedy(Solver):
    def __init__(self,bandit,init_prob=1.0):
        super().__init__(bandit)
        self.estimates = np.array([init_prob]*self.bandit.K)
        self.total_count = 0

    def run_one_step(self):
        self.total_count += 1
        if np.random.random() < 1/self.total_count:
            k = np.random.randint(0,self.bandit.K)
        else:
            k = np.argmax(self.estimates)

        r = self.bandit.step(k)

        self.estimates[k] += (r - self.estimate[k])/(self.counts[k]+1)
        return k

```
